class proto::FieldPrototype< NAME_, TYPE_, BASE_, FLAGS_, MAX_SIZE, SIZE, CONST_VALUE_, MATCHER >

Template for protocol field definition and runtime handling.

Each field has:

  • Name (enum proto::FieldName);
  • Type (T), which can be a plain type or pointer;
  • Shared buffer base pointer (BASE);
  • Flags (FieldFlags);
  • Compile-time or dynamic size;
  • Optional constant value (for fixed fields).
Template Parameters
NAME_FieldName identifier.
TYPE_Field type (plain type or pointer).
BASE_Base pointer to the shared buffer.
FLAGS_FieldFlags mask.
MAX_SIZEHard cap for runtime size (safety limit for dynamic fields).
SIZECompile-time size. Use K_ANY_SIZE for dynamic-size fields.
CONST_VALUE_Optional pointer to a constant value copied on reset/apply.
MATCHEROptional matcher callback for validation/dispatch.

Configure runtime size for dynamic fields.

Effective only when SIZE == K_ANY_SIZE. The new size is taken from size_to_set when provided, otherwise deduced from sizeof(DATA) when DATA is not a pointer and not proto::EmptyDataType. The value is clamped by MAX_SIZE. For fields with CONST_VALUE the size cannot be changed.

Template Parameters
DATAElement type used for deduction when size is not provided.
Parameters
size_to_setByte length to set, or K_ANY_SIZE to auto-deduce.
Returns
true if size accepted (or unchanged), false if it exceeds MAX_SIZE.
